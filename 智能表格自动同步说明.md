# 智能表格自动同步 → 分析结果实时更新

智能表格有新数据或变更后，用下面任一方式让本服务自动重新分析，页面上「按角色搜索」「按日期查询」就会看到最新数据。

---

## 方式一：Webhook（数据变更时立即更新）

智能表格或自动化工具在**数据变更时**请求我们的接口，我们收到后立即拉取/分析并更新。

### 接口地址

```
POST https://你的服务地址/api/ingest-from-smart-table
Content-Type: application/json
```

### 请求体（二选一）

**A. 传「数据地址」我们自己去拉取：**
```json
{
  "url": "https://智能表格的导出链接（返回 CSV 或表格文本）"
}
```

**B. 直接传表格文本（CSV/TSV）：**
```json
{
  "text": "日期,角色名称,等级,开始经验,结束经验,差值\n2026-03-01,Mao,168,10000,20000,10000"
}
```
也可用字段名 `csv` 或 `data`，内容同上。

### 可选：校验身份

在 Render / 本机环境变量里设置：

- **INGEST_SECRET** = 自定义一串密钥（如 `my-secret-123`）

请求时加上请求头：

- **X-Ingest-Secret** = 与 INGEST_SECRET 相同的值

不设置 INGEST_SECRET 时，不校验该头（谁都知道地址就能调）。

### 谁可以调这个接口

- 智能表格若支持「数据变更时调用 Webhook」，把上面的 **POST 地址** 填进去；若支持传 Body，可填 `{"url":"你的表格导出地址"}`。
- 或使用 n8n / 简道云 / Zapier 等：当智能表格更新 → 请求本接口（带 `url` 或 `text`）。

---

## 方式二：定时拉取（每 N 分钟自动更新）

不依赖 Webhook，由本服务**按间隔**去拉一次表格数据并分析。

### 配置

在 **.env**（本机）或 Render **Environment** 里添加：

```env
SYNC_URL=https://你的智能表格导出链接（每次访问返回最新 CSV/表格文本）
SYNC_INTERVAL_MINUTES=1
```

- **SYNC_URL**：必须是 GET 请求能打开、且返回表格文本（CSV/TSV）的地址。
- **SYNC_INTERVAL_MINUTES**：间隔分钟数，例如 `1` 表示每 1 分钟拉取并更新一次。

保存后重启服务（或等 Render 自动重新部署），日志里会出现类似：

`智能表格自动同步: 已开启，每 1 分钟从 SYNC_URL 拉取并更新`

---

## 小结

| 方式       | 适用场景                     | 实时程度     |
|------------|------------------------------|--------------|
| Webhook    | 智能表格/自动化支持「变更调 URL」 | 变更后立即更新 |
| 定时拉取   | 有固定导出链接、可接受分钟级延迟 | 每 N 分钟更新   |

两种方式可以只开一种，也可以同时用（例如 Webhook 为主，定时拉取做兜底）。更新后，所有人打开页面做「按角色搜索」「按日期查询」看到的都是最新分析结果。
