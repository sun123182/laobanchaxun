# 服务器智能表格 + 机器人回调说明

服务器里有一张「智能表格」，机器人通过 **URL 回调** 往里面写数据；每写一行，**可分析表格**会立刻更新，页面上按角色搜索、按日期查询会看到最新结果。

---

## 1. 机器人要请求的地址（回调 URL）

**写一行：**
```
POST https://你的服务地址/api/smart-table/row
```
例如：`https://role-exp-stats.onrender.com/api/smart-table/row`

**写多行（可选）：**
```
POST https://你的服务地址/api/smart-table/rows
```

---

## 2. 请求怎么发

- **请求头**：`Content-Type` = `application/json`
- **请求体（Body）**：JSON，字段见下表。

### 写一行（/api/smart-table/row）

| 字段（英文） | 字段（中文） | 说明 |
|-------------|-------------|------|
| roleName    | 角色名称     | 必填 |
| diff        | 差值         | 必填，数字 |
| date        | 日期         | 选填，不填用当天；支持 "3.1"、"2026-03-01" 等 |
| expStart    | 经验值(开) / 开始经验 | 选填 |
| expEnd      | 经验值(结束) / 结束经验 | 选填 |

**示例：**
```json
{
  "date": "2026-03-11",
  "roleName": "Mao",
  "diff": 5000,
  "expStart": 10000,
  "expEnd": 15000
}
```
或中文键：
```json
{
  "日期": "2026-03-11",
  "角色名称": "Mao",
  "差值": 5000,
  "经验值(开)": 10000,
  "经验值(结束)": 15000
}
```

### 写多行（/api/smart-table/rows）

Body 里传一个数组 `rows`，每一项和上面「写一行」的格式相同：

```json
{
  "rows": [
    { "date": "2026-03-11", "roleName": "Mao", "diff": 5000 },
    { "date": "2026-03-11", "roleName": "米露", "diff": 3000 }
  ]
}
```

---

## 3. 可选：加密钥（只允许机器人调）

在 Render 或本机 **Environment** 里加一条：

- Key：`SMART_TABLE_SECRET`  
- Value：你自己设的一串密码，例如 `robot123`

机器人请求时多加一个请求头：

- Key：`X-Smart-Table-Secret`  
- Value：和上面设的密码一致，如 `robot123`

不设也可以，接口照样能用。

---

## 4. 流程小结

1. 服务器内建「智能表格」：数据存在 `data/smart-table.json`。
2. 机器人有新数据时：**POST** 到 `https://你的服务地址/api/smart-table/row`（或 `/api/smart-table/rows`），Body 按上面格式。
3. 服务端收到后：把这一行（或多行）追加到智能表格，马上根据整张表重算分析结果并写入「可分析表格」。
4. 任何人打开网页：按角色搜索、按日期查询看到的都是**实时**结果。

机器人只要按这个 URL 和 Body 格式回调，就会实现「智能表格实时更新 → 可分析表格实时更新」。
